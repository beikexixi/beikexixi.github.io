Ajax跨域

发表于 2017-09-21   |   分类于 Ajax

（2017.8.14）

URL

url: scheme:://host:port/path?query#fragment

  • scheme: 协议 http/https/ftp/file
  • host: 域名或者IP地址
  • path: 路径
  • port: 端口(文件配置时1024以下的端口最好不要用)，区分网络应用程序，相同的应用程序之间交互信息
  • query: 查询字符串。参数格式: key=value&key2=value2
  • fragment: 锚点、hash哈希，定位页面的额某个位置

    同源

    协议，域名，端口均相同

跨域

不满足同源即为跨域
具备跨域资源请求能力的标签：iframe、link、script、video、audeo

解决 方法

解决方法 cors（Cross-origin resource sharing）

  • cors在移动端支持度不错，可以考虑在移动端全面尝试
  • pc上有不兼容和没有完美支持，小心踩坑
  • jsop是get形式，承载的信息量有限，所以信息量较大时cors是不错的选择

<?php
header(‘Access-Control-Allow-Origin:http://www.study.com‘);
-> 允许固定网址可访问
header(‘Access-Control-Allow-Origin:*’);
-> 允许所有访问
?>

解决方法 jsonp

  • 只适合get请求
  • jsonp利用了script标签的可跨域特性

  • 每发一次请求就创建一个Script标签，在标签中传输信息

  • 调用
    $.ajax({
    url: ‘url—‘,
    jsonp: “”, // 约定等号左边的值key(服务器根据该名称获取回调函数 )
    jsonpCallback: // 约定等号右边的value
    })

    反向代理

  • 修改配置文件
    将想要访问的网址加入到配置该虚拟域名文档中
  • 示例
    例如：当前是www.study.com中的文件，想要访问api.study.com
    配置虚拟域名的文档httpd-vhosts.conf中
    在www.study.com的配置行加入ProxyPass /api http://api.study.com
    ajax请求是url必须写成/api

    解决方法 document.domain + iframe标签

  • 适合顶级域名相同的情况
    如 www.baidu.com 和 api.baidu.com

  • 互相访问的两个html文件都设置

模板引擎artTemplate学习笔记

发表于 2017-09-21   |   分类于 前端学习笔记

-> 本问的叙述是基于artTemplate3.0，4.0版本相关的知识会明确指出

一、为什么需要artTemplate

页面大多情况下是需要根据后端数据动态渲染页面的，一般共3种方法

  • 1.字符串拼接

缺点：当需要大量字符串拼接时容易出错

  • 2.添加节点

缺点：大量操作DOM，不利于性能优化

  • 3.模板引擎(自用artTemplate)
    若ajax传回来的数据data中有属性name=”盒子”

优点：使用模板引擎渲染页面更快

artTemplate特性–摘自artTemplate官网

  1. 性能卓越，执行速度通常是 Mustache 与 tmpl 的 20 多倍（性能测试）
  2. 支持运行时调试，可精确定位异常模板所在语句（演示）
  3. 对 NodeJS Express 友好支持
  4. 安全，默认对输出进行转义、在沙箱中运行编译后的代码（Node版本可以安全执行用户上传的模板）
  5. 支持include语句
  6. 可在浏览器端实现按路径加载模板（详情）
  7. 支持预编译，可将模板转换成为非常精简的 js 文件
  8. 模板语句简洁，无需前缀引用数据，有简洁版本与原生语法版本可选
  9. 支持所有流行的浏览器

js中数组和字符串的常用方法总结

发表于 2017-09-21

数组

不改变原数组

  • slice（）
    • 用途： Array.prototype.slice.call(arguments)能将具有length属性的对象转成数组,
      此对象可以包括arguments伪数组、数组、{length:2,”0”:”haha”,”1”:’oye’}
      -》 借用call将非数组转换成数组的方法
      方法一：var args = Array.prototype.slice.call(arguments);
      方法二：var args = [].slice.call(arguments, 0);
       
var a={length:2,0:'first',1:'second'};//伪数组,有length属性
console.log(Array.prototype.slice.call(a,0));// ["first", "second"],调用数组的slice(0);

var a={length:2,0:'first',1:'second'};
console.log(Array.prototype.slice.call(a,1));//["second"]，调用数组的slice(1);

var a={0:'first',1:'second'};//去掉length属性，返回一个空数组
console.log(Array.prototype.slice.call(a,0));//[]

function test(){
  console.log(Array.prototype.slice.call(arguments,0));//["a", "b", "c"]，slice(0)
  console.log(Array.prototype.slice.call(arguments,1));//["b", "c"],slice(1)
}
test("a","b","c");

define(function(require, exports, module) {
var a = require(‘./a’)
a.doSomething()
// …
var b = require(‘./b’) // 依赖可以就近书写
b.doSomething()
// …
})

// AMD 对于依赖的模块,AMD 是提前执行

a.doSomething()
// …
b.doSomething()
// …
})

总结

1.函数模式：this指向window
2.方法模式： 函数依附于一个对象，是对象的一个属性，this指向该对象
3.构造器调用模式：即是构造函数的调用，this指向创建的对象实例
4.上下文模式：本质是对象借用不属于该对象的方法，可以利用call()和apply()、bind()实现，this的指向和上下文有关。

更改Mac电脑指定文件的额权限

发表于 2017-09-26

数字类型改变档案权限

linux档案的基本权限就有九个，分别是owner/group/others三种身份各有自己的read/write/execute权限。这九个权限是三个三个一组的。其中，我们可以使用数字来代表各个权限，各个权限的分数对照表如下：

r:4
w:2
x:1

每种身份（owner/group/others）各自的三个权限分数是需要累计的，例如当权限为: [-rwxrwx—] 分数则是:

owner = rwx = 4+2+1 = 7
group = rwx = 4+2+1 = 7
others= — = 4+2+1 = 7

所以我们设定权限的变更时，该档案的权限数字就是770了。

语法：
chmod -R 777 档案或目录
解释：
-R 进行递归的持续变更，亦即连同次目录下的所有档案都会变更

一、为什么需要artTemplate

页面大多情况下是需要根据后端数据动态渲染页面的，一般共3种方法

  • 1.字符串拼接

缺点：当需要大量字符串拼接时容易出错

  • 2.添加节点

缺点：大量操作DOM，不利于性能优化

  • 3.模板引擎(自用artTemplate)
    若ajax传回来的数据data中有属性name=”盒子”

优点：使用模板引擎渲染页面更快

artTemplate特性–摘自artTemplate官网

  1. 性能卓越，执行速度通常是 Mustache 与 tmpl 的 20 多倍（性能测试）
  2. 支持运行时调试，可精确定位异常模板所在语句（演示）
  3. 对 NodeJS Express 友好支持
  4. 安全，默认对输出进行转义、在沙箱中运行编译后的代码（Node版本可以安全执行用户上传的模板）
  5. 支持include语句
  6. 可在浏览器端实现按路径加载模板（详情）
  7. 支持预编译，可将模板转换成为非常精简的 js 文件
  8. 模板语句简洁，无需前缀引用数据，有简洁版本与原生语法版本可选
  9. 支持所有流行的浏览器

Ajax跨域

发表于 2017-09-21   |   分类于 Ajax

（2017.8.14）

URL

url: scheme:://host:port/path?query#fragment

  • scheme: 协议 http/https/ftp/file
  • host: 域名或者IP地址
  • path: 路径
  • port: 端口(文件配置时1024以下的端口最好不要用)，区分网络应用程序，相同的应用程序之间交互信息
  • query: 查询字符串。参数格式: key=value&key2=value2
  • fragment: 锚点、hash哈希，定位页面的额某个位置

    同源

    协议，域名，端口均相同

跨域

不满足同源即为跨域
具备跨域资源请求能力的标签：iframe、link、script、video、audeo

解决 方法

解决方法 cors（Cross-origin resource sharing）

  • cors在移动端支持度不错，可以考虑在移动端全面尝试
  • pc上有不兼容和没有完美支持，小心踩坑
  • jsop是get形式，承载的信息量有限，所以信息量较大时cors是不错的选择

<?php
header(‘Access-Control-Allow-Origin:http://www.study.com‘);
-> 允许固定网址可访问
header(‘Access-Control-Allow-Origin:*’);
-> 允许所有访问
?>

解决方法 jsonp

  • 只适合get请求
  • jsonp利用了script标签的可跨域特性

  • 每发一次请求就创建一个Script标签，在标签中传输信息

  • 调用
    $.ajax({
    url: ‘url—‘,
    jsonp: “”, // 约定等号左边的值key(服务器根据该名称获取回调函数 )
    jsonpCallback: // 约定等号右边的value
    })

    反向代理

  • 修改配置文件
    将想要访问的网址加入到配置该虚拟域名文档中
  • 示例
    例如：当前是www.study.com中的文件，想要访问api.study.com
    配置虚拟域名的文档httpd-vhosts.conf中
    在www.study.com的配置行加入ProxyPass /api http://api.study.com
    ajax请求是url必须写成/api

    解决方法 document.domain + iframe标签

  • 适合顶级域名相同的情况
    如 www.baidu.com 和 api.baidu.com

  • 互相访问的两个html文件都设置
